# Лендинг мультиплатформенного AI-лидгена для B2B

Одностраничный лендинг SaaS-продукта для поиска квалифицированных лидов из **16+ площадок** (Telegram, HH.ru, SuperJob, Habr Career, FL.ru, Kwork, LinkedIn, Reddit, Hacker News и др.) в формате **вайтлейбл-бренда**.

> Целевое действие: **самостоятельная регистрация в сервисе** и запуск первого проекта.

---

## 1) Цель проекта

Сделать конверсионный лендинг, который:
- объясняет ценность за 3 секунды;
- снижает страх сложности за счёт простой структуры и демонстрации интерфейса;
- ведёт пользователя к регистрации через повторяющиеся CTA;
- поддерживает мобильный сценарий как приоритетный.

**Целевая аудитория:** B2B-компании всех ниш: SMB, агентства, фрилансеры, CRM-интеграторы, маркетплейс-специалисты.

---

## 2) Позиционирование

**Не “инструмент парсинга”, а поток квалифицированных клиентов** из 16+ площадок.

Ключевой месседж:
> «Клиенты ищут вас прямо сейчас на 16+ площадках. Мы их находим.»

---

## 3) KPI лендинга

- Конверсия: **8–15%** (посетитель → регистрация)
- Время до первого скролла: **< 3 сек**
- Bounce rate: **< 40%**
- Среднее время на странице: **> 2 минут**

---

## 4) Структура лендинга (11 блоков)

1. **Hero** — ценность + главный CTA + мини-доказательство
2. **Как это работает** — 3 шага за 4 минуты
3. **Демо интерфейса** — дашборд / список лидов / чат, автосмена каждые 5 сек
4. **Ключевые преимущества** — почему лучше ручного поиска и альтернатив
5. **Кейсы с цифрами** — социальное доказательство по нишам
6. **ROI-калькулятор** — прогноз выручки и окупаемости в реальном времени
7. **Тарифы** — прозрачная матрица планов + выделение рекомендованного
8. **Сравнение с альтернативами** — таблица «ручной / ключевики / наш сервис»
9. **FAQ** — снятие возражений перед регистрацией
10. **Финальный CTA** — последний конверсионный толчок
11. **Футер** — юридические ссылки, контакты, поддержка

---

## 5) Требования к дизайну

- Стиль: современный, технологичный, чистый
- Без стоковых фото людей; только интерфейс, данные, схемы, иконки
- Референсы по духу: Linear / Vercel / Stripe
- Палитра:
  - тёмный фон: `#0A0A0F`
  - акцент: `#FF6B35`
  - позитивные метрики: `#34D399`
  - вторичный текст: `#9898AA`
- Типографика: Manrope (или аналог), акцент на крупных заголовках и цифрах

---

## 6) Технические требования

- Формат реализации: статический лендинг (HTML + Tailwind CSS + JS) или SSR-фреймворк
- Адаптивные диапазоны:
  - Mobile: 320–768 (высший приоритет)
  - Tablet: 768–1024
  - Desktop: 1024–1440
  - Wide: 1440+
- Цели по производительности:
  - Lighthouse Performance: 90+
  - FCP: < 1.5 сек
  - LCP: < 2.5 сек
- SEO:
  - семантическая разметка H1-H3
  - Meta / Open Graph / Twitter Card
  - Schema.org (SoftwareApplication + FAQPage)
  - robots.txt и sitemap.xml
- Интеграции:
  - редирект регистрации на `platform.leadl.ai` с UTM
  - Яндекс.Метрика + GA4
  - пиксель VK (опционально)

---

## 7) Контент и тон коммуникации

- Тон: прямой, конкретный, без воды
- Цифры важнее эпитетов
- Без обещаний фиксированного результата без оговорки о нише
- Для вайтлейбла: **не упоминать Leadl.ai в текстах лендинга**
- Минимизировать англицизмы без необходимости (с учётом требований ФЗ-168)

---

## 8) Конверсионная механика

### Основные CTA (4 точки)
1. Hero: «Получить первых лидов бесплатно»
2. После блока шагов: «Попробовать за 4 минуты»
3. Под ROI-калькулятором: «Начать получать этих клиентов»
4. Финальный блок: «Начать бесплатно»

### Микроконверсии для аналитики
- глубина скролла (10/25/50/75/100)
- клик по CTA
- взаимодействие с калькулятором
- раскрытие FAQ
- клик по Telegram-контакту
- время на странице >30 / >60 / >120 сек

## 12) Быстрый локальный запуск

Так как проект статический, достаточно открыть `index.html` в браузере.

Для запуска через локальный сервер (рекомендуется):

```bash
python -m http.server 8080
```

Далее открыть: `http://localhost:8080`
